# CateringOS - הקשר לפרויקט (Project Context)

## 1. מהות הפרויקט (Project Essence)
**שם הפרויקט:** CateringOS
**תיאור:** מערכת הפעלה וניהול לעסק קייטרינג, הבנויה על גבי האקוסיסטם של Google.
**מצב נוכחי:** שלב **Reboot** (אתחול מחדש). הפרויקט עבר ניקוי מלא מקוד ישן כדי להתחיל בבניית תשתית דאטה-בייס (Database) נכונה ויציבה מאפס, ללא חובות טכניים מהעבר.
**המטרה המיידית:** בניית מסד נתונים חכם ב-Google Sheets שינהל את עצי המוצר (Ingredients -> Recipes -> Dishes), ספקים, ואירועים.

## 2. הטכנולוגיה (Tech Stack)
*   **פלטפורמה:** Google Apps Script (GAS).
*   **Database & UI:** Google Sheets (השיטס משמש גם כמסד נתונים וגם כממשק למשתמש).
*   **שפה:** JavaScript (ES6).
*   **ניהול גרסאות:** Git + GitHub.
*   **פריסה (Deployment):** כלי `clasp` (לסנכרון בין הקוד המקומי לענן של גוגל).

## 3. חוקי ברזל (Iron Rules & Guidelines)
חוקים אלו מגדירים איך אנחנו עובדים (לקוח מתוך `.cursorrules`):
1.  **עברית קודם:** כל ממשק המשתמש (כותרות, סטטוסים, הודעות שגיאה) חייב להיות בעברית (RTL). הקוד הפנימי (משתנים, הערות) באנגלית.
2.  **Zero Guessing:** המערכת לא ממציאה נתונים. אם חסר מידע, הוא נשאר חסר עד להשלמה ידנית. אין המרות יחידה ללא הגדרה מפורשת.
3.  **UI/UX Constraints:** טבלאות השיטס הן לתצוגה בלבד (Display-only). עריכה מורכבת וקבלת החלטות מתבצעות דרך סרגל צד (Sidebar) ייעודי ולא בתוך התאים.
4.  **ביצועים:** שימוש ב-Batch Operations (`getValues`, `setValues`) בלבד. איסור על קריאה/כתיבה שורה-אחר-שורה.

## 4. מבנה הקבצים הנוכחי (Current File Structure)
הפרויקט כרגע **נקי** (Clean Slate) ומכיל רק קבצי קונפיגורציה ותשתית:
*   `appsscript.json`: קובץ המניפסט של גוגל (מגדיר את הפרויקט, הרשאות, ו-Timezone: Asia/Jerusalem).
*   `.cursorrules`: קובץ ההנחיות ל-AI (עודכן לשלב בניית הדאטה בייס).
*   `.claspignore` / `.gitignore`: קבצי התעלמות לתשתית.
*   **חסר (בכוונה):** אין כרגע קבצי `.js` או `.html`. כל הלוגיקה תיכתב מחדש בשלב זה.

## 5. החיבורים הפעילים (Active Integrations)
1.  **Google Cloud Project:** הקובץ `appsscript.json` מחובר לפרויקט קיים בגוגל (Script ID קיים). משמעות: כל `clasp push` מעדכן את הסקריפט החי שמחובר לגיליון האלקטרוני.
2.  **GitHub Repo:** הפרויקט מחובר ל-Repository פרטי, ענף נוכחי: `cursor/database-reset-and-rebuild`.

## 6. חזון הדאטה בייס (Database Vision - To Be Built)
זהו המבנה שאנו מתכננים לבנות כעת (הקשר ל-AI החדש):
המערכת צריכה לנהל היררכיה של פריטי מזון:
1.  **Raw Materials (חומרי גלם):** הפריט הבסיסי ביותר (קמח, עגבניה, בשר). נרכש מספקים.
2.  **Prep (הכנות):** תוצרי ביניים (רוטב עגבניות, בצק). מורכב מחומרי גלם.
3.  **Dishes (מנות):** המוצר הסופי שמוגש לאורח. מורכב מ-Prep וחומרי גלם.
4.  **Suppliers (ספקים):** מחירונים, מק"טים, ויחידות רכש.

**האתגר המרכזי:** ניהול יחידות מידה (ק"ג מול יחידות מול ארגזים) וחישוב עלויות מדויק (Costing) לאורך העץ.
